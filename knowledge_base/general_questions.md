# About This Website

## How does this chatbot work?

This chatbot is powered by a Retrieval-Augmented Generation (RAG) system. When you ask a question, it first searches my personal knowledge base for the most relevant information. That context is then provided to a Large Language Model (LLM), allowing it to generate accurate, personalized responses about my background, projects, and experience.

---

# About This Chatbot

## Why did you build this chatbot?

I built this chatbot as a hands-on learning project to deepen my understanding of Advanced RAG techniques. Rather than just reading about query rewriting, hybrid search, and re-ranking, I wanted to build a working system from scratch. I also wanted a living portfolio piece that lets people learn about me in an interactive way, rather than just reading a static resume.

---

## What tech stack did you use for this website?

The backend is built with Python and FastAPI, deployed on Railway via Docker. For retrieval, I use ChromaDB for vector search and BM25 for keyword search, combined with Reciprocal Rank Fusion. Re-ranking is handled by a CrossEncoder model, and GPT-4o-mini generates the final answers. The frontend is a simple HTML/CSS/JS site hosted on GitHub Pages.

---

## How long did it take you to build this?

I built this over the course of a few weeks, designing and implementing each module of the RAG pipeline step by step — from ingestion and retrieval to re-ranking and generation, then deploying and connecting the frontend.
